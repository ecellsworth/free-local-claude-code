# Free Local Claude Code (Ollama) — One-Click Setup

Run an AI coding assistant **entirely on your own machine** — zero cost, fully
offline, no account and no API key. This project wraps the "Claude Code + Ollama"
idea into a single double-click installer for macOS and Linux, adds the things
the typical guides leave out (preflight checks, install fallbacks, model and
system‑prompt switching, and a clean uninstaller), and fixes the mistakes those
guides commonly make.

Under the hood it uses **Ollama** (a local LLM engine) to serve an open coding
model such as `qwen2.5-coder:7b`, and **Claude Code** (Anthropic's CLI) as the
agent front‑end. Claude Code talks to Ollama through Ollama's
Anthropic‑compatible API on `localhost` — so nothing leaves your computer at
runtime.

> **What this is / isn't.** This runs an *open* model (qwen2.5-coder, deepseek-coder-v2, …) through the Claude Code interface. It is **not** hosted Claude and does not match its quality. It's great for offline, private, no‑cost coding, Markdown, and automation work — not for tasks needing the web, huge contexts, or top‑tier accuracy.

## Features

- **Preflight checks.** Detects your OS/architecture and package manager, checks
  curl, RAM, and free disk, reports what's already installed, installs what's
  missing, then a **hard gate re‑confirms** every prerequisite before the setup
  proceeds. If something can't be confirmed, it stops rather than half‑install.
- **One‑click installer.** A `.command` file you double‑click (macOS) or run in a
  terminal (Linux). Installs Ollama, Node.js, and Claude Code, pulls a local
  model, wires everything up, and writes a ready‑to‑use launcher.
- **Install fallbacks.** Every install step tries alternatives if the first
  method fails (e.g. Homebrew → cask → versioned `.dmg`; npm → sudo →
  `--unsafe-perm` → official installer).
- **Latest‑stable versions, never beta.** Looks up the newest *stable* Ollama
  release at install time and falls back to a pinned known‑good version; it
  rejects anything that looks like rc/beta/alpha/nightly.
- **Offline & zero‑cost by design.** Local models only (it refuses `:cloud`
  models), and it disables Claude Code telemetry, auto‑update, and error
  reporting so nothing phones home.
- **Model switching.** Keep `qwen2.5-coder:7b` as default and switch to
  `deepseek-coder-v2:16b` (or any local model) at startup with a single flag.
- **System‑prompt switching.** Swap the assistant's behavior between an
  offline‑tuned prompt (distilled from a full **Claude Fable 5** system prompt,
  keeping only the parts that work locally) and your own — without breaking
  Claude Code's built‑in tooling.
- **One‑click uninstall.** A `.command` that removes **only** what the setup
  installed (tracked in an install manifest) — it never touches packages that
  existed on your system beforehand, and never deletes your files.

## Requirements

- macOS or Linux. (Windows users: run inside WSL2.)
- ~5 GB free disk for a 7B model (more for larger models).
- ~8 GB RAM recommended for a 7B model; ~10–12 GB for `deepseek-coder-v2:16b`.
- Internet **for installation only**. Usage afterward is fully offline.

## Quick start

1. Download `setup-claude-ollama.command`.
2. **macOS:** double‑click it in Finder. If macOS blocks it, right‑click → **Open**
   (or run `chmod +x setup-claude-ollama.command` first).
   **Linux:** `bash setup-claude-ollama.command`
3. Follow the prompts. When it finishes, open a **new terminal** and run:

   ```bash
   claude-local
   ```

That's it — you're coding against a local model, offline, for free.

Want a different default model at install time:

```bash
MODEL="deepseek-coder-v2:16b" bash setup-claude-ollama.command
```

## All commands

### Install

```bash
# macOS: double-click setup-claude-ollama.command, or:
bash setup-claude-ollama.command

# choose a different default model
MODEL="deepseek-coder-v2:16b" bash setup-claude-ollama.command
```

### Preflight (built into the installer)

The installer runs preflight automatically. To re‑check your environment at any
time without installing, you can inspect the pieces it verifies:

```bash
uname -s; uname -m                     # OS + architecture
command -v curl node npm ollama claude # which prerequisites are present
curl http://localhost:11434/api/tags   # is the local Ollama API up?
ollama list                            # which local models are installed
```

### Run the assistant

```bash
claude-local                                   # interactive session (default model)
claude-local -p "your prompt"                  # one-shot, print result, exit
claude-local -m deepseek-coder-v2:16b          # pick a model for this run
CCMODEL=deepseek-coder-v2:16b claude-local     # same, via env var
```

### Model management

```bash
ollama pull deepseek-coder-v2:16b   # add another local model (one-time)
ollama list                         # list installed models
ollama rm <model>                   # remove a model
ollama ps                           # show running models
```

### System‑prompt switching

```bash
claude-local --append-system-prompt-file system-prompt-local-offline.md
claude-local --append-system-prompt-file CLAUDE-FABLE-5.md
claude-local                                   # no custom prompt
```

### Uninstall

```bash
# macOS: double-click uninstall.command, or:
bash uninstall.command
```

## Using the assistant (start, stop, switches)

| Action | Command / key |
|--------|---------------|
| Start an interactive session | `claude-local` |
| One‑shot prompt (no chat UI) | `claude-local -p "…"` |
| Pipe a file/command in | `cat file.py \| claude-local -p "review this"` |
| **Stop the current task / exit the session** | `ESC` or `Ctrl+C` |
| **Quit Claude back to your shell** | `Ctrl+C` (twice) |
| Continue most recent session | `claude-local -c` |
| Resume a specific session | `claude-local -r <id>` |
| Use a different model this run | `claude-local -m <model>` |
| Add custom behavior rules | `claude-local --append-system-prompt-file <file>` |
| Machine‑readable output | `claude-local -p "…" --output-format json` |
| Start the model engine (if needed) | `ollama serve` |

> Always use **`claude-local`**, not plain `claude`. Plain `claude` points at the
> paid hosted API and will fail offline ("API error / retrying").

## Use cases & prompt examples

This setup shines at offline coding, Markdown work, and automation. Start a
session with `claude-local` and try prompts like these (or run them one‑shot with
`claude-local -p "…"`).

**Coding**

- `Refactor parser.py to handle empty input and add unit tests with pytest.`
- `Find and fix the off-by-one bug in pagination.js, then run the tests.`
- `Explain what main.go does and where the HTTP routes are defined.`

**Automation / scripting**

- `Write a bash script that backs up ~/Documents to /Volumes/Backup and logs results with timestamps.`
- `Create a Python script that renames every .jpeg in this folder to .jpg.`
- `Write a cron-ready script that deletes log files older than 30 days.`

**Markdown: create, edit, parse**

- `Turn notes.txt into a clean README.md with a title, sections, and a table of contents.`
- `Read report.md and tighten it to half the length without losing key points.`
- `Extract every TODO comment in this repo into a checklist in TODO.md.`

**File conversion (uses local tools like pandoc/ffmpeg)**

- `Write a pandoc command to convert all .md files in this folder to PDF.`
- `Give me an ffmpeg command to convert input.mov to a web-friendly mp4.`

**Reading files in a session**

- `Read task.md and do exactly what it says.`  *(or reference it with `@task.md`)*

> Tip: keep work in a project folder and launch `claude-local` from there. The
> assistant can read any file in that folder and its subfolders by name.

## Model switching

Add a stronger model once (downloads, then offline forever):

```bash
ollama pull deepseek-coder-v2:16b
```

Then pick a model at startup:

```bash
claude-local                              # default: qwen2.5-coder:7b
claude-local -m deepseek-coder-v2:16b     # this run uses deepseek
```

To change the **permanent** default, edit the `MODEL=` line in
`~/.local/bin/claude-local`. Recommended local coding models:

```
qwen2.5-coder:7b   (default)   qwen2.5-coder:3b / :14b / :32b
deepseek-coder-v2:16b          codellama:13b          llama3.1:8b
```

Avoid any `:cloud` model — those run on Ollama's servers (they cost money and
require internet, defeating the purpose).

## System‑prompt switching

Claude Code accepts a file that is **appended** to its system prompt to shape the
assistant's behavior (tone, rules, style). This project ships an offline‑tuned
prompt, `system-prompt-local-offline.md`, distilled from a full **Claude Fable 5**
system prompt — keeping only the model‑ and environment‑agnostic guidance
(tone/formatting, honesty, safety, coding habits) and dropping everything tied to
hosted‑only tools that don't exist locally.

```bash
# offline-tuned behavior
claude-local --append-system-prompt-file system-prompt-local-offline.md

# the original full prompt (for comparison/experimentation)
claude-local --append-system-prompt-file CLAUDE-FABLE-5.md

# default behavior
claude-local
```

Run the command from the folder that contains the file. **Use
`--append-system-prompt-file`, not `--system-prompt-file`.** The latter *replaces*
Claude Code's built‑in instructions and breaks its file editing.

## Uninstall

```bash
# macOS: double-click uninstall.command, or:
bash uninstall.command
```

The uninstaller reads the install manifest written during setup and removes
**only** what the setup installed:

- Always: the `claude-local` launcher and the setup's block in your shell profile.
- If the setup installed them: the pulled models, Claude Code, and Ollama.
- Homebrew and Node get extra confirmation (default no), since they usually
  pre‑exist.

Anything that existed before the setup is left untouched, and your own files
(scripts, READMEs, prompt files, projects) are never deleted. If no manifest is
found (e.g. an older install), it runs in a safe interactive mode and asks about
each item.

## Troubleshooting

| Symptom | Cause / fix |
|--------|-------------|
| `API error · Retrying…` | You ran plain `claude`. Use **`claude-local`** (it sets the local endpoint). |
| `command not found: claude-local` | Open a **new** terminal, or run the full path `~/.local/bin/claude-local`. |
| `connection refused` / no response | Ollama isn't running. `ollama serve`, then check `curl http://localhost:11434/api/tags`. |
| `permission denied: ~/.zshrc` | Fix perms: `sudo chown $USER ~/.zshrc && chmod u+rw ~/.zshrc`. |
| Slow or low‑quality answers | Normal for small local models. Try `deepseek-coder-v2:16b` or a larger qwen. |
| Answers get cut off | Raise context: a model with a larger window, or increase `OLLAMA_CONTEXT_LENGTH`. |
| `claude-local` ignores `--model` | Use `-m <model>` (first argument), e.g. `claude-local -m llama3.1:8b`. |
| Want hosted Claude back | Use plain `claude` in a terminal where the local env vars aren't set. |

## How it works / FAQ

**Why `claude-local` instead of `claude`?** The launcher bakes in the offline
environment (`ANTHROPIC_BASE_URL=http://localhost:11434`,
`ANTHROPIC_AUTH_TOKEN=ollama`, plus telemetry‑off flags) and runs your local
model. Plain `claude` defaults to the hosted Anthropic API.

**Why not a proxy like claude-code-router?** Earlier guides needed a translation
proxy because Ollama didn't speak the Anthropic API. It now does, on `localhost`,
so no proxy is required.

**Is it really free / offline?** Yes. Models run on your hardware; the only
endpoint is your local Ollama server. Only the one‑time install needs internet.

**Does it work on Windows?** Use WSL2 and follow the Linux path.
